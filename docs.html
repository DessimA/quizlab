1. Início Rápido

Comece a usar o QuizLab em menos de 1 minuto.

Passo 1: Obter um Simulado

Baixe nosso modelo de exemplo (.json) ou crie o seu próprio usando o Criador Integrado.

Passo 2: Importar

Na tela inicial, arraste o arquivo para a área pontilhada ou clique em "Selecionar Arquivo".

DICA PRO: Você pode salvar até 10 simulados na sua Biblioteca Local para acesso rápido sem precisar fazer upload novamente.

2. Guia do Usuário

2.1 Realizando Testes

  • Navegação Livre: Use as setas laterais para navegar. Você pode pular questões e voltar depois.
  • Tipos de Questão:
    • Única Escolha: Apenas uma resposta correta.
    • Múltipla Escolha: Selecione exatamente o número de opções solicitadas.
  • Progresso: A barra superior indica seu avanço. O grid de questões (em breve) mostrará o status de cada pergunta.

2.2 Criando Simulados

O Criador Visual permite montar provas sem código:

  1. Preencha Título e Descrição na seção "Dados Gerais".
  2. Adicione questões uma a uma. O sistema valida automaticamente se você preencheu tudo.
  3. Use o sistema de Rascunhos Automáticos: se você fechar o navegador, seu trabalho estará salvo ao voltar (Auto-save a cada 30s).
  4. Ao finalizar, exporte o JSON e compartilhe!

2.3 Biblioteca Local

Gerencie seus estudos:

  • Busca: Encontre simulados por título ou tags.
  • Limite: Máximo de 10 itens para garantir performance máxima do navegador.

3. Referência Técnica (JSON)

Estrutura completa para criação manual:

{
  "nomeSimulado": "Título Obrigatório",
  "descricao": "Texto opcional descrevendo a prova",
  "tags": ["matematica", "dificil"], // Opcional
  "questoes": [
    {
      "id": 1, // Numérico ou String
      "enunciado": "Texto da questão <b>com HTML básico</b>",
      "tipo": "unica", // "unica" ou "multipla"
      "alternativas": [
        { "id": "a", "texto": "Alternativa 1" },
        { "id": "b", "texto": "Alternativa 2" }
      ],
      "respostasCorretas": ["a"] // Array de IDs
    }
  ]
}
ATENÇÃO: O campo respostasCorretas deve sempre ser um Array, mesmo para questões de escolha única. Os IDs devem coincidir exatamente com os IDs das alternativas.

4. Recursos & Acessibilidade

  • Teclado: Navegue usando Tab, Enter e Setas Direcionais.
  • Leitores de Tela: Interface otimizada com atributos ARIA.
  • Modo Compacto: Ideal para telas pequenas, oculta informações não essenciais.

6. Limites e Especificações

Especificação Limite Motivo
Simulados na Biblioteca 10 Performance do LocalStorage
Questões por Simulado ∞ (Rec. <500) Fluidez da Interface
Tamanho do JSON ~5MB Limite do Navegador

5. FAQ & Troubleshooting

P: Como importo vários simulados de uma vez?
R: Atualmente, importe um por vez. Isso garante a validação correta de cada arquivo.

P: O QuizLab funciona offline?
R: Sim! Após carregar a página, tudo funciona sem internet.

P: "Botão Confirmar não aparece"
R: Em questões de múltipla escolha, você deve selecionar EXATAMENTE o número de respostas corretas pedido.

P: Meu JSON dá "Erro de Formato"
R: Verifique se não há vírgulas sobrando no final de listas ou arrays. Use um validador online como jsonlint.com.
